Remarks

A dictionary class built on top of InternalSet<KeyValuePair<K,V>>.

Template Parameters
K
V

Benchmarks show that this class is not as fast as the standard Dictionary{K,V} in most cases. however, it does have some advantages:

  • MMap allows null as a key (assuming it is based on the second version of Impl.InternalSet{T}).
  • MapOrMMap{K, V}.TryGetValue and MapOrMMap{K, V}.ContainsKey do not throw an incredibly annoying exception if you have the audacity to ask whether there is a null key in the collection.
  • This class supports fast cloning in O(1) time.
  • You can convert a mutable MMap{K,V} into an immutable Map{K,V}, a read-only dictionary that does not change when you change the original MMap.
  • This class has an AddRange method.
  • This class has some bonus features: MapOrMMap{K, V}.TryGetValue(K, V) returns a default value if the key is not present; AddIfNotPresent only adds a pair if the collection does not already contain the key; AddOrFind can retrieve the current value and change it to a new value at the same time; and GetAndRemove can get a value while it is being deleted.
  • The persistent map operations Union, Intersect, Except and Xor combine two dictionaries to create a new dictionary, without modifying either of the original dictionaries. Equally interesting, the methods With and Without create a new dictionary with a single item added or removed.

The documentation of InternalSet{T} describes how the data structure works.

bool  replaceIfPresent

Adds a pair to the map if the key is not present, retrieves the existing key-value pair if the key is present, and optionally replaces the existing pair with a new pair.

Parameters
pairWhen calling this method, pair.Key specifies the key that you want to search for in the map. If the key is not found then the pair is added to the map; if the key is found, the pair is replaced with the existing pair that was found in the map.
replaceIfPresentThis parameter specifies what to do if the key is found in the map. If this parameter is true, the existing pair is replaced with the specified new pair (in fact the pair in the map is swapped with the pair parameter). If this parameter is false, the existing pair is left unmodified and a copy of it is stored in the pair parameter.
Returns
True if the pair's key did NOT exist and was added, false if the key already existed.
